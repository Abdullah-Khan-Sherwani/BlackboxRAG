AVIATION SAFETY RAG - QUICK START CHECKLIST
================================================================================

PROJECT:  NTSB Aviation Accident Report RAG System
STACK:    Pinecone + BM25 + RRF + Reranker + HF Inference API + Streamlit
EVAL:     deepeval (Faithfulness + Relevancy) + Ablation Study
DEADLINE: April 5, 2026
CORPUS:   75-100 reports sampled from 7,462 NTSB reports (2016-2023)

QUERY CAPABILITIES (single pipeline, different prompts):
  - Single-accident Q/A: "What caused ERA24LA084?" or identify by description
    (user does NOT need the NtsbNo — semantic search matches by details)
  - Cross-document synthesis: "Compare engine failures in Cessna vs Piper"
  - Follow-up reasoning: "Were there similar accidents?" triggers multi-report retrieval
  - The retriever is the SAME for all queries. Only the LLM prompt template differs.
  - Stretch goal: safety recommendation tracking (deferred unless time permits)

================================================================================
DAY 1: DATA PREP + CHUNKING + PINECONE UPSERT
================================================================================

Data Loading:
  [ ] Load CSV (sep=';', encoding='utf-8')
  [ ] Drop 15 duplicate rep_text rows
  [ ] Sample 75-100 reports (stratified by year for diversity)
  [ ] Clean text: remove "Page X of Y", normalize whitespace

Chunking Strategy A - Recursive (for ablation):
  [ ] Use RecursiveCharacterTextSplitter (chunk_size=1500, overlap=200)
  [ ] Generate chunks from sampled reports
  [ ] Attach metadata: chunk_id, ntsb_no, date, state, make, model

Chunking Strategy B - Section-Aware (for ablation):
  [ ] Split on NTSB section headers (History of Flight, Findings, etc.)
  [ ] Sub-split oversized sections at sentence boundaries
  [ ] Same metadata attachment

Embedding + Pinecone:
  [ ] Embed all chunks with all-MiniLM-L6-v2 (384 dimensions)
  [ ] Create Pinecone index: name="ntsb-rag", dim=384, metric="cosine"
  [ ] Upsert vectors + metadata in batches of 100
  [ ] Verify with a test query

Checkpoint: Chunks in Pinecone, BM25 corpus saved as pickle
Expected: 500-2000 chunks from 75-100 reports

================================================================================
DAY 2: HYBRID RETRIEVAL + RRF
================================================================================

BM25 Setup:
  [ ] Build BM25Okapi index from chunk texts (rank_bm25 library)
  [ ] Save as bm25_index.pkl (for deployment to HF Space)

Dense Retrieval:
  [ ] Query Pinecone with embedded query (all-MiniLM-L6-v2)
  [ ] Return top-K results with metadata

Reciprocal Rank Fusion:
  [ ] Implement RRF: score(d) = sum(1/(k+rank)) across both lists
  [ ] Merge BM25 and Pinecone results into single ranked list
  [ ] Test: compare hybrid results vs dense-only vs BM25-only

Checkpoint: hybrid_retrieve(query) returns fused ranked chunk IDs
